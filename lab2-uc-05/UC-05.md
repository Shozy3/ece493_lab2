## Use Case UC-05: Save Submission Draft

**Goal in Context**: An author saves paper submission information at any stage of the submission process.

**Scope**: Conference Management System

**Level**: Subfunction

**Primary Actor**: Author

**Secondary Actors**: Database

**Trigger**: Author clicks the save button during paper submission.

### Success End Condition
* Paper information is saved in database as draft, and author receives confirmation.

### Failed End Condition
* Paper information is not saved, and author is notified of validation errors if any.

### Preconditions
* User is in the submission process.
* Author has entered some information (may be incomplete).

### Main Success Scenario
1. Author clicks "Save" during submission.
2. System checks paper information validity.
3. If valid, system saves submission info as draft.
4. System confirms draft is saved.

### Extensions
* **2a**: System finds a validity violation
  * 2a1: System informs the user of the violation.
  * 2a2: System does not save invalid data.
  * 2a3: User corrects information and saves again.

### Related Information
* **Priority**: High
* **Frequency**: Frequent during submission process
* **Open Issues**: None
