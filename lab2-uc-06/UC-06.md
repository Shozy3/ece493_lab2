## Use Case UC-06: Assign Referees to Papers

**Goal in Context**: An editor assigns referees to a submitted paper by email so the paper can be reviewed.

**Scope**: Conference Management System

**Level**: User Goal

**Primary Actor**: Editor

**Secondary Actors**: Email Service, Database, Referee

**Trigger**: Editor requests referee assignment for a paper.

### Success End Condition
* Three referees are assigned to the paper, each referee receives review invitation, and system confirms assignment.

### Failed End Condition
* Assignment is not completed, and editor is informed of the issue.

### Preconditions
* Paper has been submitted.
* Editor is a registered user.
* List of available referees is available.

### Main Success Scenario
1. Editor selects a paper and enters referee email address.
2. System checks referee's assigned-paper count (must be â‰¤ 5).
3. If no violation, system sends paper information as invitation to referee email.
4. Editor repeats steps 1-3 for second referee.
5. Editor repeats steps 1-3 for third referee.
6. System confirms that three referees have been assigned.

### Extensions
* **2a**: Reviewer already has more than 5 assigned papers
  * 2a1: System informs editor of violation.
  * 2a2: Assignment not completed, editor selects different referee.
* **3a**: Referee receives invitation and may reject
  * 3a1: System records rejection if referee rejects invitation.
  * 3a2: Editor may assign different referee.

### Related Information
* **Priority**: High
* **Frequency**: Frequent during review assignment period
* **Open Issues**: None
