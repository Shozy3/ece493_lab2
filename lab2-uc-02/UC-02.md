## Use Case UC-02: User Login

**Goal in Context**: A registered user logs into the system to access CMS features.

**Scope**: Conference Management System

**Level**: User Goal

**Primary Actor**: Registered User

**Secondary Actors**: Authentication Service, Database

**Trigger**: User requests to log in to the system.

### Success End Condition
* User is authenticated and redirected to their home page with appropriate role-based access.

### Failed End Condition
* User remains on login page with error message, no access granted.

### Preconditions
* User is registered in the system.
* System is operational.

### Main Success Scenario
1. User requests log-in.
2. System asks for username and password.
3. User provides username and password.
4. System compares provided credentials with stored credentials in database.
5. System authenticates user.
6. System redirects user to their home page.

### Extensions
* **4a**: Credentials do not match
  * 4a1: System shows error message indicating invalid credentials.
  * 4a2: User may retry with correct credentials or proceed to registration.

### Related Information
* **Priority**: High
* **Frequency**: Very frequent
* **Open Issues**: None
