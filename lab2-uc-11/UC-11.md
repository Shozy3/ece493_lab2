## Use Case UC-11: Edit and Update Schedule

**Goal in Context**: An editor edits the schedule so the final version is updated and distributed.

**Scope**: Conference Management System

**Level**: User Goal

**Primary Actor**: Editor

**Secondary Actors**: Database, Email Service, Audience, Web Page

**Trigger**: Admin/editor requests edits to the schedule.

### Success End Condition
* Schedule is updated, new version replaces old version, schedule is sent to audience, and announced on CMS webpage.

### Failed End Condition
* Schedule is not updated, old version remains, and editor is notified of failure.

### Preconditions
* A schedule exists (has been generated).

### Main Success Scenario
1. User requests schedule edits.
2. System allows editor to edit/update schedule.
3. System replaces prior schedule with the new final version.
4. System sends final schedule to audience.
5. System announces schedule on CMS webpage.

### Extensions
* None specified in the SRS text.

### Related Information
* **Priority**: High
* **Frequency**: Occasional (may happen multiple times before conference)
* **Open Issues**: None
