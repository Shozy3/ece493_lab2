## Use Case UC-12: Pay for Conference Attendance

**Goal in Context**: An authorized user pays for conference registration by credit card to receive a ticket.

**Scope**: Conference Management System

**Level**: User Goal

**Primary Actor**: Authorized User

**Secondary Actors**: Payment Gateway, Database, Email Service

**Trigger**: Authorized user requests conference registration.

### Success End Condition
* Payment is processed, registration is confirmed, ticket is stored in system and sent to user via email.

### Failed End Condition
* Registration is not completed, payment is not processed, and user is notified of failure.

### Preconditions
* User is authorized (registered).
* Conference registration is open.

### Main Success Scenario
1. Authorized user requests conference registration.
2. System provides credit-card payment interface.
3. User completes payment.
4. System stores payment confirmation and sends it to user as a ticket.

### Extensions
* **3a**: Payment is declined by payment gateway
  * 3a1: System displays error message indicating payment failure.
  * 3a2: User may retry with different payment method or contact support.
* **4a**: Database error occurs during storage
  * 4a1: System displays error message (payment may have been processed).
  * 4a2: User contacts support to verify registration status.

### Related Information
* **Priority**: Medium
* **Frequency**: Frequent during registration period
* **Open Issues**: Payment gateway integration details not specified
