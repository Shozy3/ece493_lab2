## Use Case UC-08: Submit Review Form

**Goal in Context**: A referee fills and submits the review form for an assigned paper so the editor can decide.

**Scope**: Conference Management System

**Level**: User Goal

**Primary Actor**: Referee

**Secondary Actors**: Database, Editor

**Trigger**: Referee selects review action for an assigned paper.

### Success End Condition
* Review form is completed, saved in database, and sent to editor. When all three reviews are complete, editor can make decision.

### Failed End Condition
* Review is not submitted, form data may be lost, and referee must retry.

### Preconditions
* Referee has accepted invitation and has access to assigned paper in their account.

### Main Success Scenario
1. Referee logs in and selects review action for an assigned paper.
2. System redirects to review page with the review form.
3. Referee fills the form and submits.
4. System validates form fields (no invalid characters, no blanks).
5. System saves the review and sends it to the editor.
6. System checks if all three reviewers have submitted reviews.
7. If all reviews complete, system notifies editor that paper is ready for decision.

### Extensions
* **4a**: Form has invalid characters/blanks
  * 4a1: System fails validation.
  * 4a2: Referee must correct form before submission.

### Related Information
* **Priority**: High
* **Frequency**: Frequent during review period
* **Open Issues**: None
