## Use Case UC-01: Register New User

**Goal in Context**: A user registers a new account in the system to gain access to CMS features.

**Scope**: Conference Management System

**Level**: User Goal

**Primary Actor**: User

**Secondary Actors**: Database, Email Validation Service

**Trigger**: User requests to be registered on the system.

### Success End Condition
* User account is created with unique email address, password is stored securely, and user is redirected to login screen.

### Failed End Condition
* No account is created, user remains on registration form, and error message is displayed.

### Preconditions
* User is not yet registered with the email they intend to use.
* System is operational and accessible.

### Main Success Scenario
1. User requests to be registered.
2. System redirects to registration form.
3. User fills and submits the form with email, password, and other required information.
4. System validates email/password fields.
5. System verifies email is unique and valid.
6. System stores user information in database.
7. System redirects user to login screen.

### Extensions
* **4a**: Email/password fields invalid
  * 4a1: System shows error message indicating invalid fields.
  * 4a2: User corrects fields and resubmits.
* **5a**: Email not unique or not valid
  * 5a1: System displays error message indicating email is already registered or invalid format.
  * 5a2: User enters different email or proceeds to login if already registered.

### Related Information
* **Priority**: High
* **Frequency**: Frequent during conference registration periods
* **Open Issues**: None
